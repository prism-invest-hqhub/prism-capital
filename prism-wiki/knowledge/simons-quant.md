# 西蒙斯量化

数据驱动的概率优势——棱镜的第五层执行框架。

---

## 核心理念

> "我不猜测市场方向，我只寻找统计优势。" —— 詹姆斯·西蒙斯

- 不依赖主观判断，依赖数据统计
- 每一笔交易都有明确的概率和期望值
- 高频次 × 正期望 = 长期必胜

## 量化分析流程（L0-L6）

```
L0 数据采集 → L1 清洗去噪 → L2 指标计算 → L3 策略信号
    → L4 风控过滤 → L5 仓位计算 → L6 执行输出
```

### 各节点说明

| 节点 | 输入 | 处理 | 输出 |
|------|------|------|------|
| L0 | 腾讯/新浪API | 实时行情拉取 | 原始行情数据 |
| L1 | 原始数据 | 去除停牌/异常/ST | 清洗后数据 |
| L2 | 清洗数据 | 双低/PE/溢价率计算 | 量化指标 |
| L3 | 量化指标 | 筛选条件匹配 | 候选标的 |
| L4 | 候选标的 | 止损/仓位/纪律过滤 | 可执行标的 |
| L5 | 可执行标的 | 凯利公式计算仓位 | 具体金额 |
| L6 | 仓位信号 | 通知/建议 | 最终输出 |

## 棱镜API量化能力

| 端点 | 数据源 | 说明 |
|------|--------|------|
| `/realtime` | 腾讯(主)+新浪(备) | A股/ETF实时行情 |
| `/index` | 腾讯 | 核心指数行情 |
| `/bond/double-low` | efinance+腾讯 | 可转债双低排名 |

## 置信度标准

- **禁止拍脑袋给置信度**——必须有推导过程
- 每个环节有具体数字支撑
- 禁止输出无依据的虚假精确结果
- 估算必须标注「估算」+明确依据

---

> 量化不是万能的，但不量化是万万不能的。所有结论必须有数据支撑。
